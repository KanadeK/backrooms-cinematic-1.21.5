# Contributing

Do not submit copyrighted textures, audio, maps, screenshots, private logs, UUIDs, server addresses, credentials or packaged third-party mods without documented permission. Every asset contribution needs a source, author, licence, modification status and a row in `docs/ASSET_PROVENANCE.md`.

Use focused changes, retain the release gates, run the relevant Python validation commands, and describe any runtime or multiplayer evidence precisely. Do not claim a test that was not performed. By contributing original code or documentation, you agree that it may be licensed under MIT; asset licences are handled separately.
